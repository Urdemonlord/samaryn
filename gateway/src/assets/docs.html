1. Overview

Samaryn is an OpenAI-compatible reverse proxy that sits between your application and LLM providers. Every request is inspected before being forwarded — prompt injection is blocked, PII is masked, and traffic is routed to the provider you choose.

Key capabilities

Block   Stop prompt injection, jailbreak attempts, and out-of-domain requests. Mask   Redact emails, phone numbers, NIK, and other sensitive patterns. Route   Forward requests to any upstream provider through a single endpoint.

2. Quick Start

The fastest way to test Samaryn is with cURL. You'll need an API key from your Samaryn deployment operator.

Base URL

https://samaryn.meowlabs.id

Simple test

curl https://samaryn.meowlabs.id/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "kr/claude-sonnet-4.5",
    "messages": [
      {"role": "user", "content": "Balas singkat: sistem aktif"}
    ]
  }'

Health check

curl https://samaryn.meowlabs.id/health

Expected response: {"service":"samaryn-gateway","status":"healthy","version":"0.2.0"}

3. Authentication

All API requests require an API key sent via the Authorization header.

Header format

Authorization: Bearer <YOUR_API_KEY>

Getting an API key

API keys are managed by your Samaryn operator. Check the Dashboard or contact your deployment admin. The following endpoints are public and do not require authentication:

  • GET / — Landing page
  • GET /demo — Interactive demo
  • GET /docs — This documentation
  • GET /health — Health check

4. Chat Completions

The /v1/chat/completions endpoint follows the OpenAI chat completions format. Your existing OpenAI client code works by changing the base URL and API key.

Endpoint

POST https://samaryn.meowlabs.id/v1/chat/completions

Request body parameters

Parameter Type Required Description
model string yes Model identifier. Use a route name (e.g. kr/claude-sonnet-4.5) or any model ID the upstream provider supports.
messages array yes Array of message objects with role and content.
stream boolean no Set to true for streaming responses via SSE. Default: false.
temperature number no Sampling temperature (0–2). Default: 1.
max_tokens integer no Maximum tokens in the response. Default and limits depend on the upstream model.

Response format

The response follows the OpenAI chat completions format, with additional security-related fields: 

{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "created": 1234567890,
  "model": "kr/claude-sonnet-4.5",
  "provider": "meowlabs",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Sistem aktif."
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 45,
    "completion_tokens": 12,
    "total_tokens": 57
  },
  "security": {
    "flagged": false,
    "pii_redacted": false,
    "injection_detected": false,
    "decision": "allow"
  }
}

The security object shows what the gateway detected and what decision was made.

5. Model Routing

Samaryn can route requests to multiple upstream providers. Use the model field to select the route.

Available routes

To list all available routes:

curl https://samaryn.meowlabs.id/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"

Route format

Models are identified by route prefix and model name:

Route Example model Provider
kr/ kr/claude-sonnet-4.5 MeowLabs Knowledge Router
kr/ kr/deepseek-3.2 MeowLabs DeepSeek route
kr/ kr/glm-5 MeowLabs GLM route
kr/ kr/qwen3-coder-next MeowLabs Qwen route
gemini/ gemini/gemini-2.5-flash Direct Gemini

You can also pass any model ID the upstream provider supports — Samaryn forwards it transparently.

6. Security Features

Every request is evaluated by Samaryn's policy engine before reaching the upstream model. The result is returned in the security response field.

Prompt injection detection

Samaryn detects attempts to override system instructions, extract prompts, or manipulate the model into unsafe behavior. Injection attempts are blocked before the request reaches the provider.

BLOCKED   Requests flagged as prompt injection return a 403 with a clear error message.

PII masking

Personally identifiable information (email, phone numbers, NIK, tokens, API keys) can be automatically redacted or masked from requests. This is configured at the gateway level.

MASKED   Sensitive data is replaced with placeholders before forwarding.

Policy decisions

Decision HTTP status Description
allow 200 Request passed all checks
block 403 Prompt injection detected
mask 200 Allowed but PII was redacted
escalate 200 Out-of-domain context, passed with warning

7. Code Examples

Samaryn uses the OpenAI-compatible format, so any OpenAI SDK works by changing the base URL.

Python (openai SDK)

from openai import OpenAI

client = OpenAI(
    base_url="https://samaryn.meowlabs.id/v1",
    api_key="YOUR_API_KEY"
)

response = client.chat.completions.create(
    model="kr/claude-sonnet-4.5",
    messages=[{"role": "user", "content": "Balas singkat: sistem aktif"}]
)

print(response.choices[0].message.content)

Python (httpx — raw)

import httpx

response = httpx.post(
    "https://samaryn.meowlabs.id/v1/chat/completions",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "model": "kr/claude-sonnet-4.5",
        "messages": [{"role": "user", "content": "Balas singkat: sistem aktif"}]
    }
)

print(response.json()["choices"][0]["message"]["content"])

JavaScript (fetch)

const response = await fetch("https://samaryn.meowlabs.id/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model: "kr/claude-sonnet-4.5",
    messages: [{ role: "user", content: "Balas singkat: sistem aktif" }]
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);

cURL (streaming)

curl https://samaryn.meowlabs.id/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "kr/claude-sonnet-4.5",
    "messages": [{"role": "user", "content": "Hitung 1 sampai 5"}],
    "stream": true
  }'

8. Error Handling

Samaryn returns standard HTTP status codes with JSON error bodies in the OpenAI error format.

Error response format

{
  "error": {
    "message": "Description of what went wrong",
    "type": "error_type",
    "code": "http_status_code"
  }
}

Common error codes

Status Type Description
400 bad_request Invalid request body or missing required field
401 authentication_error Missing or invalid API key
403 blocked_by_security_policy Request blocked by security policy (prompt injection, PII threshold)
404 not_found_error Unknown endpoint or model
500 upstream_error Upstream provider error or gateway failure
502 bad_gateway Upstream provider unreachable

Blocked request example

{
  "error": {
    "message": "Request blocked by security policy: prompt injection detected",
    "type": "blocked_by_security_policy",
    "code": "403"
  },
  "security": {
    "flagged": true,
    "injection_detected": true,
    "decision": "block"
  }
}
