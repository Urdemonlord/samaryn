1. Overview & Architecture

Samaryn adalah model deteksi prompt injection berbasis IndoBERT yang dibungkus dalam gateway OpenAI-compatible. Setiap request yang masuk melewati pipeline keamanan tiga lapis sebelum diteruskan ke LLM tujuan.

Pipeline arsitektur

Layer 1 — Rule Engine: Pola eksplisit seperti instruksi override, exfiltration, keyword ofensif, dan marker injection dikenal dengan latency hampir nol. Request yang cocok langsung diblok tanpa memanggil model.

Layer 2 — IndoBERT Classifier: Request yang lolos rule engine diklasifikasi oleh model IndoBERT ONNX ke dalam tiga kelas: safe, prompt_injection, atau out_of_domain.

Layer 3 — Policy Decision: Berdasarkan output model, policy engine menentukan tindakan: allow, block, mask, atau escalate. Keputusan dicatat untuk audit trail.

Cloud vs Self-Host

Samaryn bisa dipakai sebagai cloud endpoint di samaryn.meowlabs.id atau di-self-host via Docker. Model IndoBERT sudah terbundle dalam image — tinggal jalankan dan hubungkan ke LLM atau chatbot yang ingin diamankan. Keduanya menggunakan model deteksi yang identik.

2. The IndoBERT Model

IndoBERT adalah varian BERT yang di-pre-train secara khusus untuk Bahasa Indonesia. Samaryn melakukan fine-tuning IndoBERT untuk tugas klasifikasi tiga kelas, dengan output berupa probabilitas untuk setiap kelas.

Skema klasifikasi

Kelas Label Deskripsi
safe allow Request normal dalam domain — diteruskan ke LLM tujuan.
prompt_injection block Mengandung instruksi manipulatif, override, atau eksfiltrasi — diblokir.
out_of_domain escalate Request di luar domain sistem — ditandai untuk review atau log.

Dataset dan training

Model dilatih pada dataset campuran:

  • In-domain benign prompts — prompt normal berbahasa Indonesia
  • Prompt injection samples — diadaptasi dari benchmark publik seperti BIPIA, InjecAgent, LLMail-Inject
  • Out-of-domain samples — input di luar distribusi domain target

Model di-export ke ONNX runtime untuk inference dengan latency rendah di production.

3. Quick Start — Cloud API

Cara tercepat menguji Samaryn adalah lewat cloud endpoint. Anda memerlukan API key dari operator deployment.

Base URL

https://samaryn.meowlabs.id

Simple test

curl https://samaryn.meowlabs.id/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "user", "content": "Balas singkat: sistem aktif"}
    ]
  }'

Health check

curl https://samaryn.meowlabs.id/health

Expected response: {"service":"samaryn-gateway","status":"healthy","version":"0.2.0"}

5. Chat Completions API

Endpoint /v1/chat/completions mengikuti format OpenAI chat completions. Sebelum request diteruskan ke LLM tujuan, request discan oleh rule engine dan IndoBERT classifier.

Endpoint

POST https://samaryn.meowlabs.id/v1/chat/completions

Request body parameters

Parameter Type Required Description
model string yes Model ID LLM tujuan (e.g. gpt-4o-mini). Request discan oleh IndoBERT sebelum diforward ke model ini.
messages array yes Array message objects with role and content.
stream boolean no SSE streaming. Default: false.
temperature number no Sampling temperature (0–2). Default: 1.
max_tokens integer no Maximum tokens. Default dan limit bergantung pada upstream model.

Response format

Response mengikuti format OpenAI chat completions, dengan tambahan field security yang menunjukkan hasil scan model IndoBERT: 

{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "created": 1234567890,
  "model": "gpt-4o-mini",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Sistem aktif."
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 45,
    "completion_tokens": 12,
    "total_tokens": 57
  },
  "security": {
    "flagged": false,
    "pii_redacted": false,
    "injection_detected": false,
    "decision": "allow"
  }
}

Field security menunjukkan apa yang dideteksi oleh model IndoBERT dan keputusan policy engine.

6. Self-Host Deployment

Samaryn bisa di-deploy di infrastruktur sendiri via Docker. Model IndoBERT ONNX sudah terbundle dalam image — tidak perlu download model terpisah.

Prerequisites

  • Docker & Docker Compose
  • Server dengan CPU (minimal 4 core, 8GB RAM) — GPU tidak wajib untuk ONNX inference
  • API key untuk upstream LLM yang ingin diamankan

Quick start

git clone https://github.com/Urdemonlord/samaryn.git
cd samaryn
cp config/default.yaml config/production.yaml
# Edit config/production.yaml — set API keys dan endpoint
docker compose up -d

Konfigurasi

File konfigurasi config/default.yaml berisi pengaturan untuk gateway dan ml-service. Yang perlu disesuaikan:

  • Upstream LLM API keys — untuk forwarding request setelah discan
  • Threshold classifier — batas keputusan untuk tiap kelas
  • Rule engine patterns — daftar pola eksplisit yang ingin dicegat
  • Audit log destination — file atau endpoint eksternal

Arsitektur container

Docker Compose menjalankan dua service:

  • gateway — Rust binary di port 8080. Menangani routing, rule engine, dan forwarding.
  • ml-service — Python FastAPI di port 8000. Menjalankan model IndoBERT ONNX.

7. Security Decisions

Setiap request dievaluasi oleh rule engine dan IndoBERT classifier sebelum mencapai upstream model. Hasil evaluasi tersedia di field security pada response.

Prompt injection detection

Model IndoBERT mendeteksi upaya override instruksi sistem, ekstraksi prompt, dan manipulasi model ke perilaku tidak aman. Request yang terdeteksi sebagai injection diblokir sebelum mencapai provider.

BLOCKED   Request yang diblokir mengembalikan HTTP 403 dengan pesan error yang jelas.

Out-of-domain detection

Tidak semua request mencurigakan adalah serangan. Samaryn membedakan prompt injection dari request yang cuma berada di luar domain sistem. OOD request bisa dieskalasi untuk review tanpa diblokir sepenuhnya.

ESCALATE   Request out-of-domain diteruskan dengan status escalated dan dicatat untuk audit.

Policy decisions summary

Decision HTTP status Description
allow 200 Request lolos semua pemeriksaan — diteruskan ke LLM tujuan
block 403 Prompt injection terdeteksi oleh model IndoBERT atau rule engine
mask 200 Aman tetapi PII direduksi sebelum diferuskan
escalate 200 Out-of-domain context — diteruskan dengan peringatan

8. Error Handling

Samaryn mengembalikan HTTP status codes standar dengan JSON error bodies dalam format OpenAI error.

Error response format

{
  "error": {
    "message": "Description of what went wrong",
    "type": "error_type",
    "code": "http_status_code"
  }
}

Common error codes

Status Type Description
400 bad_request Invalid request body or missing required field
401 authentication_error Missing or invalid API key
403 blocked_by_security_policy Request diblokir oleh IndoBERT classifier atau rule engine
404 not_found_error Unknown endpoint or model
500 upstream_error Upstream LLM provider error or gateway failure
502 bad_gateway Upstream provider unreachable

Blocked request example

{
  "error": {
    "message": "Request blocked by security policy: prompt injection detected",
    "type": "blocked_by_security_policy",
    "code": "403"
  },
  "security": {
    "flagged": true,
    "injection_detected": true,
    "decision": "block"
  }
}
