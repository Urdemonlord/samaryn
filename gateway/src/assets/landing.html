Model IndoBERT sebagai engine deteksi utama

Samaryn menggunakan IndoBERT — varian BERT yang di-pre-train untuk Bahasa Indonesia — sebagai classifier tiga kelas. Setiap request dinilai sebelum diteruskan ke LLM, dengan hybrid pipeline yang menggabungkan rule engine, model, dan escalation layer.

IndoBERT fine-tuned untuk deteksi prompt injection

Model dilatih pada dataset campuran: prompt normal berbahasa Indonesia, sampel injection dari benchmark publik (BIPIA, InjecAgent, LLMail-Inject), dan sampel out-of-domain. Output berupa probabilitas tiga kelas.

  • Fine-tuning IndoBERT untuk klasifikasi safe / injection / OOD
  • Inference via ONNX runtime untuk latency rendah
  • Threshold tuning untuk menyeimbangkan false positive dan false negative

Tiga kelas keputusan, bukan sekadar blocklist

Kebanyakan detector cuma binary: safe atau block. Samaryn membedakan prompt injection berbahaya dari request yang cuma berada di luar domain — dua situasi yang butuh penanganan berbeda.

  • Safe → allow dan forward ke LLM
  • Prompt injection → block sebelum mencapai model
  • Out-of-domain → escalate untuk review atau log

Hybrid pipeline: rule engine + model + escalation

Model tidak bekerja sendirian. Rule engine menangkap pola eksplisit dengan latency hampir nol, model menangani kasus yang lebih kompleks, dan escalation layer menangani ambiguitas.

  • Hard rules untuk override, exfiltration, intent ofensif
  • IndoBERT ONNX classifier sebagai fallback tiga kelas
  • Escalation path untuk request yang tidak jelas aman atau berbahaya

Cloud API atau self-host di infrastruktur sendiri

Samaryn bisa dipakai sebagai cloud endpoint atau di-deploy sendiri via Docker. Model IndoBERT sudah included dalam image — tinggal jalankan dan hubungkan ke LLM atau chatbot yang ingin diamankan.

  • Cloud: satu endpoint OpenAI-compatible, tinggal ganti base URL
  • Self-host: Docker Compose, model ONNX sudah terbundle
  • Cocok untuk tim yang butuh kontrol data dan compliance

Alur deteksi: dari request ke keputusan

Setiap request melewati pipeline tiga lapis — rule engine, IndoBERT classifier, dan policy decision — sebelum diteruskan ke LLM tujuan. Hasilnya adalah keputusan yang bisa diaudit.

01

Request masuk

Aplikasi mengirim request dalam format OpenAI-compatible ke endpoint Samaryn, menyertakan model LLM yang ingin dipakai dan prompt pengguna.

02

Rule engine

Pola eksplisit diperiksa lebih dulu: override, exfiltration, intent ofensif, dan marker injection dikenal. Jika cocok, request langsung diblok tanpa panggil model.

03

IndoBERT classifier

Jika lolos rule engine, request dikirim ke model IndoBERT ONNX untuk klasifikasi tiga kelas: safe, prompt injection, atau out-of-domain.

04

Decision + forward

Berdasarkan output model: safe → forward ke LLM, injection → block 403, OOD → escalate. Semua keputusan dicatat untuk audit trail.
