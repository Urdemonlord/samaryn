Fokus ke proteksi, bukan kosmetik dashboard

Samaryn dirancang untuk tim yang butuh guardrail nyata di jalur request LLM. Surface kecil, keputusan jelas, dan integrasi tetap sederhana.

Policy engine yang memutuskan sebelum model dipanggil

Setiap request diperiksa lebih dulu. Jika pola prompt injection atau data sensitif terdeteksi, gateway bisa block, redact, atau warn sesuai policy.

  • Rule-based scan untuk pola eksplisit dan serangan yang sudah dikenal
  • ML service terpisah untuk klasifikasi tambahan dan fallback keamanan
  • Audit trail terstruktur agar keputusan mudah dilacak

Drop-in untuk aplikasi yang sudah pakai API OpenAI

Endpoint tetap familiar. Kamu tinggal ganti base URL lalu route ke provider yang diinginkan dari gateway yang sama.

  • Support OpenAI-compatible request flow
  • Bisa route ke OpenAI, Anthropic, Gemini, atau OpenRouter
  • Cocok untuk self-hosting di VPS dan container

Bahasa Indonesia bukan afterthought

Positioning Samaryn kuat untuk use case lokal yang sering luput saat hanya mengandalkan filter bahasa Inggris.

  • Prompt dan contoh demo sudah pakai Bahasa Indonesia
  • PII masking relevan untuk format data lokal
  • Roadmap hybrid scan diarahkan ke akurasi konteks Indonesia

Operational shape yang realistis buat production

Arsitektur dipisah jelas antara gateway dan scanner. Ini memudahkan scaling, restart service, dan observability di lingkungan server yang sederhana.

  • Gateway Rust untuk path utama dan streaming
  • ML service Python untuk scan yang lebih berat
  • Docker Compose ready untuk deployment cepat

Alur request yang mudah dipahami tim engineering

Tujuannya sederhana. Semua traffic masuk ke satu titik, diperiksa, lalu diteruskan ke provider bila lolos policy.

01

Ingress tunggal

Aplikasi mengirim request ke Samaryn dengan format yang tetap familiar bagi client OpenAI-compatible.

02

Security scan

Gateway menjalankan rules engine, PII detection, lalu meminta klasifikasi tambahan ke ML service bila dibutuhkan.

03

Decision

Policy menentukan apakah request diizinkan, dimasking, atau dihentikan sebelum mencapai upstream model.

04

Forward + audit

Request yang lolos diteruskan ke provider pilihan dan keputusan dicatat untuk observability.
